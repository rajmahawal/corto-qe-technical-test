# Test Reporting

This guide explains the reports and debugging artifacts generated by Playwright.

For test execution commands, refer to [Running Tests](running-tests.md).

## HTML Report

Playwright generates an HTML report after test execution.

Open the latest report with:

```bash
npm run report
```

The report is useful for reviewing:

- Passed and failed tests
- Test duration
- Assertion failures
- Screenshots, videos, and traces when available

## Generated Artifact Locations

Playwright generated output is stored in:

```text
playwright-report/
test-results/
```

These folders are ignored from Git because they are generated during test execution.

## Trace Viewer

The framework retains traces on failure.

Open a trace using:

```bash
npx playwright show-trace path/to/trace.zip
```

Trace Viewer helps investigate:

- Page actions
- Locator resolution
- Network calls
- Console output
- Screenshots at each step
- Timing issues

## Screenshots and Videos

Screenshots and videos are retained on failure.

They are useful for UI E2E debugging because they show the browser state and user journey at the time of failure.

## API Failure Diagnostics

API assertion helpers provide additional diagnostic information when a status assertion fails.

The error message includes:

- Expected status
- Actual status
- Request URL
- Response body

This makes API failures easier to debug from terminal output or CI logs.

## CI Artifacts

In CI/CD, the following paths are uploaded as artifacts when available:

```text
playwright-report/
test-results/
```

These artifacts allow failures to be investigated without reproducing the issue locally.

## Local Debugging Flow

When a test fails locally:

1. Read the terminal error message.
2. Open the HTML report.
3. Check screenshots or videos if available.
4. Open the trace if deeper investigation is needed.
5. Fix the issue and rerun the targeted test.
